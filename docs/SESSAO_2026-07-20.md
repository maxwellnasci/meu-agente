## Pendência fechada: backup git das extensões próprias

Problema: `ask-max`, `whatsapp-cloud`, `response-audit` e
`github-repo-report` viviam só dentro de `openclaw/extensions/` —
`openclaw/` é um clone git de terceiros (remote
`github.com/openclaw/openclaw`), inteiro ignorado por
`meu-agente/.gitignore`. Essas 4 extensões, código próprio, nunca
tiveram backup real em lugar nenhum. O único registro de parte delas
era o branch local `production-local-fixes` dentro do `.git` interno do
`openclaw/` — nunca enviado a lugar nenhum.

**Por que não dava pra simplesmente "adicionar" essas pastas ao git do
`meu-agente`:** `openclaw/` tem seu próprio `.git`. Git não permite
rastrear seletivamente uma subpasta de um repositório aninhado a partir
do repo pai — ele só aceita "engolir" o repo aninhado inteiro como
ponteiro de submódulo (SHA de commit), nunca o conteúdo real dos
arquivos.

**Por que não usamos symlink** (mover os arquivos pra fora e deixar um
link simbólico no lugar): o Amigão roda em produção via Docker
(`openclaw-openclaw-gateway-1`), com `Dockerfile`/`docker-compose.yml`
dentro de `openclaw/` e `COPY . .` cujo contexto de build é a própria
pasta `openclaw/`. Um symlink apontando pra fora dela ficaria quebrado
dentro da imagem gerada, derrubando a extensão em produção.

**Solução aplicada — cópia de exportação (snapshot, não histórico):**

1. Criado `meu-agente/extensions/` (pasta nova, irmã de `openclaw/`).
2. Copiado o conteúdo atual das 4 pastas de `openclaw/extensions/*`
   pra lá — `openclaw/extensions/*` permanece intocada, zero risco pro
   container rodando.
3. `meu-agente/.gitignore` não mudou — `openclaw/` continua ignorada
   inteira, sem exceções (evita o problema do repo aninhado).
4. Criado `scripts/sync-extensions-backup.sh`: refaz a cópia sob
   demanda (via `rsync`, exclui `node_modules`/`dist`/`.env`/`*.log`)
   pra quando o código dessas extensões mudar dentro de
   `openclaw/extensions/` no futuro. Passo manual, não automático —
   escolhido de propósito por simplicidade (repo pessoal, sem CI).
5. Decisão sobre histórico: **snapshot único**, sem preservar os
   commits que já existiam no `.git` interno do `openclaw/`
   (`production-local-fixes`). Objetivo era backup de segurança do
   estado atual, não histórico de como foi construído — a
   documentação em `docs/` já cobre o "como e por quê" de cada decisão.
6. Commit e push reais confirmados: `03bee75..f668374 master -> master`
   em `github.com/maxwellnasci/meu-agente`.

**Estado final:** as 4 extensões próprias têm agora backup real e
verificável no GitHub. `openclaw/extensions/*` continua sendo a fonte
de verdade em produção (nada mudou no runtime); `extensions/*` no
`meu-agente` é o backup, atualizado sob demanda com o script.
