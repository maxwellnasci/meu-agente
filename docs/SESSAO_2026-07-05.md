## Problema conhecido identificado
nginx-app-1 não sobe automaticamente após reboot.
Causa provável: race condition com chatwoot-rails-1 
(ambos usam porta 3000 internamente).
Solução futura: adicionar "depends_on: chatwoot" 
no docker-compose do nginx-app-1.

## Procedimento pós-reboot (documentar!)
Após qualquer reboot do servidor Contabo:
1. ssh contabo
2. uname -r (confirmar kernel novo)
3. docker ps (verificar containers)
4. Se nginx-app-1 não aparecer:
   docker start nginx-app-1
   sleep 30 && docker ps | grep nginx-app-1

## Lições aprendidas
1. apt upgrade não derruba containers Docker
   (Docker atualiza sem parar os serviços)
2. Kernel novo só ativa após reboot
3. "unless-stopped" não garante subida após reboot
   se há conflito de porta na inicialização
4. Verificar TODOS os containers após reboot
